## 🎯 Why This Exists

Global supply chains move **$26 trillion** in cargo annually. A single typhoon can delay 400+ vessels, costing **$billions in demurrage**. Existing tools are reactive — they tell you what went wrong **after** it happened.

**Smart Supply Chain AI** uses Gemini 2.0's **Function Calling API** to act as an autonomous agent that detects disruptions, queries live data, and reroutes vessels — **before** losses cascade.

## 🚀 Quick Start (Local)

### Prerequisites
- **Python 3.10+** and **Node.js 20+**
- A free [Gemini API Key](https://aistudio.google.com/app/apikey)

### 1. Clone
```bash
git clone https://github.com/kirancodes-dev/SmartSupplyChain.git
cd SmartSupplyChain/smart-supply-chain
```

### 2. Backend
```bash
cd backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
echo 'GEMINI_API_KEY="your_key_here"' > .env
uvicorn main:app --reload --port 8000
```

### 3. Frontend
```bash
cd ../frontend
npm install
npm run dev
```

### 4. Open
```
http://localhost:3000/home  →  Landing page
http://localhost:3000/dashboard  →  Operations center
http://localhost:8000/docs  →  FastAPI Swagger UI
```

> **No API key?** The app runs in full mock mode — all features work with simulated AI responses.

## ☁️ Deploy to Google Cloud Run

```bash
# 1. Backend
cd smart-supply-chain/backend
gcloud run deploy smart-supply-backend \
  --source . --region us-central1 --allow-unauthenticated \
  --set-secrets "GEMINI_API_KEY=GEMINI_API_KEY:latest"

# 2. Frontend (replace URL)
cd ../frontend
gcloud run deploy smart-supply-frontend \
  --source . --region us-central1 --allow-unauthenticated \
  --set-env-vars "NEXT_PUBLIC_API_URL=https://YOUR_BACKEND_URL/api,NEXT_PUBLIC_WS_URL=wss://YOUR_BACKEND_URL/ws"
```

See [cloud_deployment_guide.md](./cloud_deployment_guide.md) for full step-by-step instructions.
